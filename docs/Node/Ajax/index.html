Ajax

​ Ajax 这项技术由来已早，是一项旧的技术，并且这个名称已经过时了；

​ AJAX 是一个简写，全称是

​ A asynchronous 异步的 [eɪˈsɪŋkrənəs]

​ J JavaScript

​ A and

​ X XML 指的一种标记语言，用于传输数据，这种标记语言已经过时了。

​

Ajax 的介绍​

​ ajax 这项技术已经非常成熟了，只是用于数据交换的格式已经发生了变化

​ 从传统的 xml 格式，变成了 json 格式，但是叫做 ajaj 不好听，所以还是 ajax

​ xml 格式的写法已经过时了，早期在前后端交互的时候，使用以下格式传输数据

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<Books>
    <item>
        <image src="https://192.168.11.22:300/uploads/aaa.jpg" />
        <name>三国演义</name>
        <price>29.8</price>
    </item>
    <item>
        <image src="https://192.168.11.22:300/uploads/bbb.jpg" />
        <name>水浒传</name>
        <price>19.8</price>
    </item>
</Books>

​ 以前的做法，就是获取到这些数据之后，使用传统的 dom 方法，把假的 dom 中的数据

​ 移植到真实的 dom 中，后端利用这种数据格式中的内容，标签，属性等等传递数据

​ 目前流行的方式，更加轻量，好操作一些

[
  { "id": 2423, "name": "迪丽热巴", "age": 22 },
  { "id": 4353, "name": "古力娜扎", "age": 21 }
]

ajax 的其他介绍​

​ ajax 是异步请求的统称，其内部的原理是 XMLhttpRequest 构造器实现的。

​ 这是原生的构造器，也是由 XML 开头的，但是实际上跟 XML 没关系；

​

​ ajax 的原生，步骤有些复杂，使用情景也比较多；（不要求记忆封装的 ，只需要记住流程）

​ ajax 既然由来已久，那么关于封装它的工具也是一大堆，所以没有必要自己再封装一遍

​ 将来企业级的开发，使用的是更高级的技术...

Ajax 的使用​

​ 以往的请求，只能靠 form， 标签的 href 属性或者 src 属性...

​ 最直接的方式，就是 url 地址栏也会发送请求

​ 大部分都是 get 请求，并且在请求的时候，大部分会跳转页面，小部分是同步要等待，阻塞

原生请求​

​ 使用 ajax 的请求，解决了异步请求，局部刷新的问题

// 1. 创建异步请求的对象 - 大驼峰的方式
const xhr = new XMLHttpRequest();

// 2. 配置请求行信息
// 参数1： 请求方式   参数2：请求路径
xhr.open("get", "http://192.168.23.6:5000/data");

// 2.1 监听请求的事件
xhr.onreadystatechange = function () {
  // 监听的其实是前端请求状态的改变
  // 0 实例创建了
  // 1 请求行已经配置了，还未发出
  // 2 请求已经发出了
  // 3 请求已经完成了，部分数据已经回来了
  // 4 请求结束，所有的数据接收完毕   --  监听的其实是这个
  if (xhr.readyState !== 4) return;

  // 能执行这里，证明是4，不然就被上面的return给结束了
  // 上面判断的都是客户端的请求状态，还需要判断服务器端的响应状态码
  //  1xx  2xx  3xx  4xx 5xx
  if (xhr.status === 200) {
    // 能执行到这里，证明前后端都没问题了，请求也结束了，服务器端也ok
    console.log(xhr.response);
  }
};

// 3. 发送请求 - 并携带请求体
// get 请求没有请求体，要么不写，要么写一个null， post请求默认需要传入 urlencoded 类型
xhr.send(null);

// 初步封装
function getData(method, url) {
  const xhr = new XMLHttpRequest();
  xhr.open(method, url);
  xhr.onreadystatechange = function () {
    if (xhr.readyState !== 4) return;
    if ((xhr.status >= 200 && xhr.status < 300) || xhr.status === 304) {
      const data = JSON.parse(xhr.response);
      console.log(data);
    }
  };
  xhr.send(null);
}
function getData(method, url, query) {
  // 1. 创建实例对象
  const xhr = new XMLHttpRequest();

  // 2. 编辑请求行信息
  let str = "?";
  if (query && method.toUpperCase() === "GET") {
    for (const k in query) {
      // k 是键，query[k]  是值
      // 如果查询参数有中文，尽量转为 ASCII 码
      str += k + "=" + encodeURI(query[k]) + "&";
    }
    // 一定要少一位
    str = str.slice(0, -1);
    // 处理 url 的拼接
    url += str;
  }

  xhr.open(method, url);

  // 2.1 监听请求的状态
  xhr.onreadystatechange = function () {
    if (xhr.readyState !== 4) return;
    if ((xhr.status >= 200 && xhr.status < 300) || xhr.status === 304) {
      const data = JSON.parse(xhr.response);
      console.log(data);
    }
  };

  let body = null;
  if (method.toUpperCase() === "POST") {
    body = "";
    for (const k in query) {
      // k 是键，query[k]  是值
      // 如果查询参数有中文，尽量转为 ASCII 码
      body += k + "=" + encodeURI(query[k]) + "&";
    }
    body = body.slice(0, -1);
  }
  // console.log(body);
  // 3. 发送请求携带请求体
  xhr.send(body);
}

function getData(config) {
  // 1. 创建实例对象
  const xhr = new XMLHttpRequest();
  // 提前解构，方便使用
  const { method = "get", url, query, data, success } = config;

  // 2. 编辑请求行信息
  let str = "?query=" + +new Date();
  if (query && method.toUpperCase() === "GET") {
    for (const k in query) {
      // k 是键，query[k]  是值
      str += "&" + k + "=" + encodeURI(query[k]);
    }
  }

  xhr.open(method, url + str);

  // 2.1 监听请求的状态
  xhr.onreadystatechange = function () {
    if (xhr.readyState !== 4) return;
    if ((xhr.status >= 200 && xhr.status < 300) || xhr.status === 304) {
      const data = JSON.parse(xhr.response);
      success && success(data);
    }
  };

  let body = null;
  if (method.toUpperCase() === "POST") {
    body = "";
    for (const k in data) {
      // 如果查询参数有中文，尽量转为 ASCII 码
      body += k + "=" + encodeURI(data[k]) + "&";
    }
    body = body.slice(0, -1);
  }
  // 3. 发送请求携带请求体
  xhr.send(body);
}

跨域​

​ 尽管 ajax 可以请求很多的第三方的资源，但是不代表着 ajax 可以任意的访问或者获取别人服务器的数据，如果在没有声明开放的情况下，获取其他服务器的资源会导致跨域错误.

跨域的介绍​

​ 跨域不是访问不到数据，而是请求成功了，服务器的响应也成功了，但是浏览器基于同源策略，有防御的机制，不允许异步请求携带第三方的数据。

​ 目的：防止恶意窃取其他服务器资源，也防止本站的项目受到攻击；

​ 时机： 协议，域名，端口不同，都会产生跨域的问题.

方案 1 JSONP​

​ jsonp 就是以 json 作为内部填充的方式交流数据；

​ 原理： 浏览器自带的标签如果发送的请求是外链不会受同源策略的影响。

// 真正的跟js相关的，可以获取数据的只有 script 标签
// 思路： script 标签来发请求；
/*
    因为 script 有天然的跨域的能力，但是携带回来的数据不好获取
    
    如果不封装
    a. 请求太多不好控制
    b. 同时发出影响请求进度，导致太慢
    c. 所有的接口都裸露在页面上，其他人可以随便获取
*/
// 封装方法
function getData(config) {
  const { url, success, callback } = config;
  // 准备一个script标签，并且配置上src和url
  const script = document.createElement("script");
  script.src = url + "?cb=" + callback;
  document.body.appendChild(script);
  // 准备一个回调函数 (最好是全局函数)
  window[callback] = function (res) {
    // console.log(res);
    success && success(res);
    // 在执行结束之后，移除这些东西
    delete window[callback];
    script.remove();
  };
}
// 调用方法发送请求
getData({
  url: "http://127.0.0.1:3000/users",
  callback: "fn",
  success: function (res) {
    console.log("获取数据", res);
  },
});
// JSONP jQuery 也有封装，所以可以直接使用
// 因为是动态创建的script标签，所以只能发送get请求
// 默认的标签只有form才能发post请求，再要么是xhr
// 但是xhr会引发跨域的问题，所以...

$("#btn").click(function () {
  // 发起跨域的JSONP请求
  $.ajax({
    url: "http://127.0.0.1:3000/users",
    // 内部就会动态创建script标签发请求
    dataType: "jsonp",
    success(res) {
      console.log(res);
    },
  });
});

Cookie 和 Session​

Cookie​

服务器端创建的 Cookie，响应给客户端，浏览器自动存，到规定的期限自动销毁 在期限以内，后续的所有的请求，浏览器都会给请求头自动携带所有的 Cookie

服务器端创建 Cookie

// 如果登录成功，则需要使用凭证记录并且响应回去
// 参数1：键  参数2：值，参数3： 其他信息(有效期)
res.cookie("isLogin", true, { maxAge: 30 * 1000 });
res.cookie("user", username, { maxAge: 40 * 1000 });

res.send({
  code: 200,
  msg: "登录成功！",
});

​ 验证是否登录解构字段

// 2. 验证是否登录的请求
router.get("/checklogin", (req, res) => {
  // 处理这次的请求，要求前端必须携带凭证，没有凭证跳转到登录页
  // console.log(req.cookies);

  // 2.1 如果登录成功，会在cookies中记录 login 信息
  const { isLogin } = req.cookies;
  if (isLogin) {
    // 登录成功过
    res.send({ code: 200, isLogin: isLogin });
  } else {
    // 没有登录
    res.send({ code: 500, isLogin: isLogin });
  }
});

Express 项目中的中间件

// 引入第三方中间件
var cookieParser = require("cookie-parser");

// 注册中间件
app.use(cookieParser());
数据量不能太大，单条最多4kb，最多50条，数据量越大，请求加载越慢
明文显示，不太安全，如果涉及到敏感数据，则不好处理，容易被劫持

Session​

Session 会比 Cookie 相对的安全一些，但是写起来也会相对麻烦一些，主要麻烦的还是后端

Session 能存储的内容更多一些，存储在服务器上或者数据库中，也需要返回一个凭证

既然也需要返回凭证，那么也需要使用 Cookie

下载中间件 $ npm i express-session -S

// 1. 引入 session 中间件
const session = require("express-session");

// 2. 注册 session 中间件
app.use(
  session({
    // 将来给客户端的那个键
    name: "s_id",
    // 设置加密的字符
    secret: "guigu100",
    // 是否为每次请求设置Cookie
    saveUninitialized: false,
    // Session 是否每次重新保存
    resave: true,
    // 关于 Cookie 的
    cookie: {
      // 只能在 http 请求中使用
      httpOnly: true,
      // 过期时间
      maxAge: 30 * 1000,
    },
  })
);
// 登录成功之后的处理

// 就要创建 session 的 id，以 Cookie 的形式返回
req.session.isLogin = true;
req.session.user = req.body;
res.send({
  code: 200,
  msg: "登录成功！",
});

获取和对比是否登录

// 2. 验证是否登录的请求
router.get("/checklogin", (req, res) => {
  // 直接打印session  req.session
  console.log(req.session);

  // 2.1 如果登录成功，会在cookies中记录 login 信息
  const { isLogin, user } = req.session;
  if (isLogin) {
    // 登录成功过
    res.send({ code: 200, isLogin: isLogin, msg: `欢迎您, ${user.username}` });
  } else {
    // 没有登录
    res.send({ code: 500, isLogin: isLogin });
  }
});

cookie 和 session 的区别​

区别就在于直接存储 Cookie 会导致数据都在浏览器存储，不太安全

如果数据都存储在服务器，通过 Cookie 返回的只是一个加密之后的字符而已。
