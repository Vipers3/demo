Express 框架

快速的搭建服务器，并且制作接口，连接数据库。

Express 介绍和使用​

传送门: https://www.expressjs.com.cn/

服务器端的快速开发框架，基于原生的 Nodejs 封装的，能快速启动一个服务器。

$ npm init -y 创建包管理文件 （要求项目目录不能叫做中文）

$ npm i express -S 在项目中先下载

在项目的根目录中，创建 app.js 文件，如果修改了之后一定要重启项目

// 1. 引入模块
const express = require("express");

// 2. 创建服务器
const app = express();

// 2.1 请求接口示例, 回调函数有两个参数
// req: request 请求    res: response 响应
app.get("/", (req, res) => {
  // send 方法是基于end方法封装的高级的响应方法
  res.send("ok");
});

// 3. 运行并指定端口挂载服务器
app.listen(5000, function () {
  console.log("服务运行在 http://127.0.0.1:5000.");
});

使用 $ node app.js 运行服务器

如果需要自动保存并重启服务器则使用 $ nodemon 之后回车

静态资源托管​

传统的原生的托管文件的方式比较繁琐，需要大量的判断

在 Express 中，内置的中间件处理了文件托管的功能

// 2.2 使用内置方法直接托管前端项目静态资源
// use 使用中间件   express.static 静态资源目录
app.use(express.static("./public"));

二级路由​

其实以后的业务，无非也是增删改查

对于用户来说，或者对于商品来说，或者对于订单来说功能是一样的

如果为了方便管理路由，不在 app.js 中大量书写代码，就要使用单独的模块去管理路由，也就是要把路由抽离出去，目的是为了方便编写和维护；

a. 命名更加方便， b. 书写更加方便

// 取名   add   增加   delete 删除   edit  编辑   list  查询
//  /deluser     /editGoods      /orderslist   /editorder
// 以上取名字的方式，很麻烦，很容易造成误会，后端压力也大

//   使用以下这种方式命名路由的地址，会好维护和联调一些
//   使用两级的写法，就是二级路由
//   /user/add   /user/del    /user/edit   /user/list
//   /goods/add  goods/del    /goods/edit  /goods/list

//   以上是对于命名上的一个区分, 如果是以请求方式作为区分
//   传统的方式，以命名作为功能的区分
//   /user/add   /user/del    /user/edit   /user/list
//     post         get           post        get
//   新的接口的风格  restful 风格的api
//   /users      /users       /users       /users
//    post        delete       put          get

中间件​

中间件已经在上述的操作中出现了，并且大量的使用了

在官网中的描述，中间件一共有 5 种

// 1. 唯一内置的中间件
app.use(express.static('./public'))

// 2. 路由级别中间件
app.use('/', require('./routes/index.js'))
app.use('/users', require('./routes/users.js'))

// 3. 错误处理中间件
app.use(function(err, req, res, next) {
  // 设置一些信息只有在 development 模式里面才有作用
  res.locals.message = err.message;
  const  = req.app.get('env') === 'development'
  res.locals.error = bool ? err : {};

  // 渲染对应的错误处理页面
  res.status(err.status || 500);
  res.render('error');
});

// 4. 第三方中间件 ... 可能会有很多，但是都需要下载
// $ npm i cookie-parser -S
var cookieParser = require('cookie-parser');
app.use(cookieParser());

// 5. 应用级别的中间件
app.use(function(req, res, next) {

  /*
    req:    request 请求对象
    res:    response 响应对象
    next:   下一个 - 管道方法
  */

  next();
});

中间件是一个函数，必须要通过 app.use 注册​

中间件，在客户端发起请求时，路由最后监听并且响应数据，

在响应之前，如果需要对 req 对象或者 res 对象进行一些提前的操作，那么就可以使用中间件进行拦截。

next 函数就是在拦截之后，起到放行并且传参的效果。

// 5. 创建一个应用级别中间件
// 作用，监听到post请求之后，获取请求体的内容再放行
app.use((req, res, next) => {
  if (req.method === "GET") return next();

  // 接收post请求的内容
  let str = "";
  // 5.1 注册data事件，接收回调的形参
  req.on("data", (chunk) => {
    str += chunk;
  });
  // 5.2 注册end事件，证明数据接收完毕
  req.on("end", () => {
    // 5.3 给 req.body 绑定数据
    req.body = str;
    // 完成之后成功放行
    next();
  });
});
const qs = require("qs");

// post请求
route.post("/data", (req, res) => {
  // 解析 post 请求体的字符为一个对象
  console.log(qs.parse(req.body));

  res.send("数据展示成功!");
});

需求：完成登录成功和失败的不同结果

<!-- action是提交地址  method是提交方式  name属性是将来解构的键 -->

  <p>用户名： <input type="text" name="username" /></p>

// 1. 登录的验证
router.post("/login", (req, res) => {
  // 1.1 获取用户登录的信息
  // console.log(req.body);
  const { username, password } = req.body;

  // 1.2 验证用户登录的信息是否为指定的内容
  if (username !== "admin") {
    // 1.3 根据验证结果返回不同的内容
    res.send({ code: 500, msg: "用户名不存在！" });
    return;
  } else if (password !== "abc123") {
    res.send({ code: 502, msg: "密码错误!" });
    return;
  } else {
    res.send({ code: 200, msg: "登录成功", token: "asdadassdadasdsadsadsa" });
  }
});
