fetch 和 axios

fetch​

​fetch 方法是 es6 原生的方式，走的不是 XMLHttpRequest 方式

​fetch 支持 Promise 的写法，返回 Promsie 的实例，可以通过 then 方法返回响应内容，但是第一个 then 方法需要返回格式响应格式

a. 原生的自带的请求方式，不需要引入任何其他的第三方插件
b. fetch 方法会返回一个Promise的实例，可以调用 then 方法
c. fetch 的第一个then方法获取到的响应对象必须要调用一个格式方法返回
d. 在第二个then方法中可以获取到真实的数据

注意事项：
a. 如果服务器端返回的状态码即使是404或者500，fetch也不会报错
b. 只有当网络故障请求被阻止时才会被标记为reject，被catch捕获
c. 默认不会自动携带Cookie, 需要手动携带
d. 如果直接使用，第一个then默认处理不了业务，还需要额外的封装
fetch("http://localhost:5100/books")
  .then((data) => {
    // 第一个then方法需要返回响应体的格式
    return data.json();
  })
  .then((res) => {
    console.log(res);
  });
// 带配置的请求方式
fetch("http://127.0.0.1:5100/books", {
  method: "post",
  headers: {
    "content-type": "application/json",
  },
  body: JSON.stringify({
    name: "西游记",
    price: 118.7,
  }),
})
  .then((data) => {
    return data.json();
  })
  .then((data) => {
    console.log(data);
  });
// GET 和 DELETE 不需要设置 body 或者 header 的
fetch("http://127.0.0.1:5100/books", {
  method: "get",
})
  .then((data) => {
    // 如果指定的是纯文本
    return data.text();
  })
  .then((data) => {
    console.log(data);
  });

axios​

​ 本来是个小工具，后来在其他框架的合并下逐渐流行，很火热

​ 是一个专门的 http 库 / 网络请求库， 前后端都可以使用

​ 响应体在返回的对象 data 中，不是直接给的

1. 基于Promise封装
2. 体积更小，压缩后不到30k，比较轻量
3. 前端和后端（Node.js）都可以使用
4. 有丰富的配置项(超时，请求头，拦截器...)
5. 更加安全的配置，比传统的方式能力更强

axios 配置​

​ 在发送请求的时候，axios 可以提供很多的配置项

全局配置​

​ 在一个项目中，如果所有的请求配置都是差不多的，那么可以集中声明

​ 请求地址，请求超时的配置，请求头的配置

// 1. 全局配置
// defaults 默认的配置对象
// baseURL  请求的基本路径
// 意义：发请求只需要后面的一截路径就可以
axios.defaults.baseURL = "http://localhost:3000";
// 设置请求超时 - 超时会自动断开
axios.defaults.timeout = 5 * 1000;
// 设置请求头 - header s  数据只能是英文 - 可以配置多项
axios.defaults.headers.common["chenwei"] = "abcdefg";
// axios.defaults.headers.common['Authorazition'] = 'Brken adsadasdasdasdasda'
(async function () {
  const { data: res1 } = await axios("/data");
  console.log(res1);
})();
如果在一个项目中后端接口的位置一致，那么全局没毛病，但是接口很多在全局设置的配置就不太实用​

实例配置​

​ axios 提供了一种方式，可以创建一个单独的实例

​ 从而不会影响其他 axios 请求的发送，内部请求使用的是实例

// 1. 实例配置
// create 方法会产生一个axios的实例，不会污染全局的配置
const kris = axios.create({
  // 配置实例私有的属性
  // 地址
  baseURL: "http://192.168.14.62:3000",
  // 设置超时响应
  timeout: 1 * 1000,
  // 携带请求头
  headers: {
    demo: "xyz",
    Authorazition: "asdasdaasdasd",
  },
});
// 2. 以实例的方式发请求
kris.get("/").then((res) => {
  console.log(res.data);
});
kris.get("/data").then((res) => {
  console.log(res.data);
});
// 3. 以上的东西，跟axios的实例没有关系
axios.get("http://127.0.0.1:5100/books").then((res) => {
  console.log(res.data);
});

拦截器​

​ 类似于中间件，处理的都是关于请求和响应的操作，服务器端处理

app.use(function (req, res, next) {
  // req 请求报文的对象
  // res 响应报文生成的对象

  // 放行并携带数据
  next();
});

​ axios 的拦截器，也是针对于请求和响应的操作，客户端处理

// 1. 拦截器
// interceptor   请求和响应  => interceptors
// 1.1 请求拦截
// 参数是一个函数，使用use方法挂载/注册
axios.interceptors.request.use((config) => {
  // 这里所有关于请求的事情，都是通过 config 参数完成
  // 也可以完成一些基础的配置
  // 超时的，基础路径的，请求头的
  config.baseURL = "http://127.0.0.1:3000";
  // config.timeout = 1 * 1000
  config.headers = { demo: "asdasdasd" };

  // 额外的两个场景
  // 开始发请求的时候完成 NProgress 的进度条展示
  // 登录请求不需要token，但是如果有token就需要在请求头携带

  // ☆☆☆☆☆ 一定要return出去，不然会请求不了
  return config;
});

// 1.2 响应拦截
axios.interceptors.response.use((res) => {
  // 响应成功之后，进度条就可以关闭了
  // 正常axios请求回来的数据，都被包裹在了对象的data中
  // ☆☆☆☆☆  同样，一定要return出去
  // return res
  return res.data;
});
// 2. 发送请求
axios.get("/data").then((res) => {
  // 如果响应拦截把数据返回了，那么不需要再 .data 了
  console.log(res);
});

取消请求​

// 方式1
// signal 信号，示意
// controller.abort()  取消
const Controller = new AbortController();
// signal 信号，示意
btn1.onclick = function () {
  axios
    .get("http://localhost:3000/data", {
      // 由控制器提供一个信号
      signal: Controller.signal,
    })
    .then((res) => {
      console.log(res.data);
    });
};
btn2.onclick = function () {
  // 控制器，调用取消的方法
  Controller.abort();
};

// 方式2
// cancelToken: 派发token
// source.cancel()   取消
const CancelToken = axios.CancelToken;
// 获取生成token的资源
const source = CancelToken.source();
btn1.onclick = function () {
  axios
    .get("http://localhost:3000/data", {
      // cancelToken 生成的随机字段
      cancelToken: source.token,
    })
    .then((res) => {
      console.log(res.data);
    });
};
btn2.onclick = function () {
  // source.cancel()
  // 自定义提示的内容.
  source.cancel("some msg....");
};

axios 的封装​

​ 以后的企业项目，都需要模块化，组件化，也就是当前的文件如果只写一次，那么其他的同事还得再写

​ 不是在功能上有任何封装的动作，基于 axios 会进行二次封装，目的是为了更加方便的调用，和处理配置信息

// 1. 配置信息
// baseURL   timeout    headers
// 1. 创建请求的实例
const service = axios.create({
  // 基本路径 - 公共路径
  baseURL: "http://localhost:3000",
  // 请求超时
  timeout: 10 * 1000,
  // 请求头
  headers: {
    // 请求头携带的内容
    demo: "random string",
  },
  // 携带一些公共字段 - 查询参数
  params: {
    app_id: "qjwkhneqigkkujok",
    app_secret: "VjNDVmIrZFkvaGhNdElPbmQxQzRhUT09",
  },
});

// 2. 设置请求和响应的拦截器
// 2.1 请求拦截器
service.interceptors.request.use(
  (config) => {
    // 2.1.1 每次请求都会开启进度条的展示
    NProgress.start();

    // 2.1.3 携带其他的请求头内容
    const token = localStorage.getItem("token");
    config.headers["Authorazition"] = token;

    return config;
  },
  // 统一处理请求拦截的错误
  (err) => {
    return Promise.reject(err);
  }
);

// 2.2 响应拦截器
service.interceptors.response.use(
  (res) => {
    // 关闭进度条的提示
    NProgress.done();

    // 根据响应的内容判断是否拥有结果，然后给不同的组件提示

    return res.data;
  },
  (err) => {
    // 关闭进度条的提示
    NProgress.done();
    return Promise.reject(err);
  }
);
