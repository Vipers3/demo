http协议

1. HTTP 协议​

​ http 协议是浏览器的一个应用协议，还有一个叫做 https

​ http（超文本传输协议 hyper text transfer protocal）， 基于 TCP/IP 之上的一个应用层协议

​ SMTP 邮件传输协议，FTP 文件传输协议， 都是其他软件的应用协议

​ HTTP 协议主要针对于浏览器的使用

IP 协议
    ip 就是计算机的地址，ip协议是底层协议
    规则就是 由4个 0 - 255 之间的号段，打点组成    192.168.33.9
    作用就是 可以根据唯一的这个地址，通过信号源准确找到需要交流的计算机

TCP协议
    tcp 协议也是一个底层协议，是传输协议
    tcp 就会建立三次握手，
            客户端发起请求，携带一些数据
            服务器收到客户端发起的请求，接收到了数据，然后返回一些新的数据
            客户端接收到数据之后，再次发起请求，携带返回的数据和新的数据
        为什么是三次，三次是最短的有效的沟通次数。

        传输内容

        当内容传输完毕之后，会断开连接，进行4次挥手
            客户端先告诉服务器，数据请求结束，接收完毕，返回字段
            服务器知晓请求，准备开始断开
            客户端主动断开连接
            服务器断开连接

        这些连接的事项和动作，都是内部处理的，无法查看和预览

http 是应用层协议，上述的两个协议已经在内部封装好了。

1.1 http 的应用​

​ http 是一个短连接应用，一次请求就会有一次响应，多次请求就会有多次响应

​ 比如在线聊天系统、这些，就属于长链接

请求报文​

​ 在客户端发起请求的时候，通常会形成请求报文

​ 请求行

​ 请求方式 请求 url http 协议和版本

​ 请求头

​ 客户端信息，携带的字段, 请求的 MIME 类型

​ 空行 - 用于区分头 和请求体

​ 请求体

​ 请求需要携带的参数或者数据，GET 没有请求体

​ get 请求的请求参数，在 url 地址栏中声明

响应报文​

​ 在服务器端返回响应结果的时候，也会形成响应报文

​ 响应状态行

​ 协议和版本 响应状态码 响应短语

​ 响应头

​ 缓存的控制，Cookie 的写入，返回的响应体的类型...

响应体

​ 空行 - 区分响应头和响应体

​ 核心的数据

2. ajax 语法和封装​

​ 在浏览器中默认发请求的方式有

地址栏，a标签，img标签，link标签，script标签，audio，video，iframe...
js 中的 location.href
form表单，只有form表单才可以发起post请求

2.1 url 组成​

​ url 全称，全球资源统一定位符

https://www.baidu.com:80/it/?w=690&h=414#list

​ 协议： http:// https://

​ 域名/IP： www.baidu.com 192.168.45.22

​ 端口： http 协议默认 80 作为端口，可以省略， https 默认是 443

​ 虚拟路径： 有可能是真实的，也有可能是虚拟的，还有后面的文件名称

​ 查询参数： 发起请求所携带的数据

​ 哈希值： #list 锚链接 #/add 哈希路由

2.2 域名和服务器​

​ 按道理来说，互联网中的服务器都会有自己的一个 ip 地址

​ 如果需要请求和访问，应该要根据 ip 地址来访问，如果网络资源比较多，服务器地址也会比较多，ip 地址的记忆成本就增加了，所以不好记，也不方便记

​ 所以域名就是用来管理 ip 的，简化记忆和访问的操作

​ 域名是由美国的一家公司专门管理的，中国是由工信部在管理的，一般普通人无法在工信部处理备案，由工信部授权的企业才可以，找二级供应商就需要花钱，还不一定能买到

​ 地址栏搜索的时候，输入的是域名，为什么也能直接找到服务器呢？

​ 当地址栏开始域名搜索的时候，先进行的是网络运行商 DNS 的服务器解析，把域名解析为对应的 ip 返回之后，再根据 ip 找到的服务器建立连接

​ 服务器：

​ 服务器就是一台计算机，一般来说商用的服务器要求计算能力更强一些

​ 一般情况下，学习阶段可以自己把自己的计算机部署为服务器，商用的时候，都是租赁

​ 端口：

​ 一个服务器只允许一个项目有点儿浪费，多个项目又不能同时运行

​ 所以不同的项目运行在不同的端口就可以做到区分了

2.3 异步请求​

​ 传统的方式大部分只能发送 get 请求，只有 form 表单才可以发送 post 请求

​ 不管是啥请求，大部分都是同步的，会跳转页面的，会阻塞代码，会影响用户体验

// 1. 创建异步对象
const xhr = new XMLHttpRequest();

// 2. 编辑请求行
xhr.open("GET", "url路径");

// 2.1 监听响应变化的事件
xhr.onreadystatechange = function () {
  // 2.2 判断前端的状态  0 1 2 3 4
  if (xhr.readyState !== 4) return;
  // 2.3 判断响应状态码  1xx  2xx  3xx  4xx  5xx
  if ((xhr.status >= 200 && xhr.status < 300) || xhr.status === 304) {
    // 2.4 接收到响应体   response   responseXML
    console.log(xhr.responseText);
  }
};

// 3. 发送请求
xhr.send(null);

2.4 请求参数​

get 请求

// 2. 编辑请求行
xhr.open("GET", "url路径?k=v&k=v");

// 3. get请求send方法一定是为空
xhr.send(null);

post 请求

// 2. 编辑请求行
xhr.open("POST", "url");

// 3. post请求把请求数据放到请求体中
// 建议加上请求头，表示请求体的数据格式 MIME 类型
xhr.setRequestHeader("Content-Type", "application/x-www-form-urlencoded");
xhr.send("k=v&k=v");

xhr.setRequestHeader("Content-Type", "application/json");
xhr.send(JSON.stringify({ id: 20, page: 10, size: 5 }));

// 上传文件的方式
xhr.setRequestHeader("Content-Type", "multipart/formdata");
xhr.send(文件的内容);
// 在浏览器的地址栏输入url之后回车，发生了什么？

1. 输入的url会经由运营商的DNS服务器解析为对应的ip
2. http协议中的ip协议会根据返回的ip地址找到对应服务器
3. http协议中的tcp协议会跟服务器建立3次握手
4. http发送请求，服务器响应数据
5. 数据发送完毕，客户端接收完毕，则进行tcp的4次挥手
6. 客户端的浏览器根据不同的引擎，渲染页面和解析js

2.5 请求的种类​

​ GET

​ 大部分请求，浏览器本身就是在执行浏览，所以文件的请求或者网页的初始加载都是 get 请求

​ get 可以带参数, 在地址栏后面 ?k=v&k=v ， 会直接显示在地址栏上，明文传输

​ get 请求所携带的参数比较少，有规定只能在 1024kb 以内

​ POST

​ 添加数据，或者做一些注册登录等请求，会比 get 请求相对来说更加安全一些

​ 请求数据在请求体中，没有数据大小的限制

​ 请求的时候，数据的类型有 3 种

​ 以往的项目，使用这两种请求方式就足够了 - 增删查改，不代表不使用了

​ 现在大多的项目，已经都改为了 restful 风格的 api

​ 好处在于，使用不同的请求处理增删查改，而不用纠结路由路径的名称

​ PUT

​ 修改的请求，类似于 POST，也会有请求体

​ DELETE

​ 删除的请求，类似于 GET，也是没有请求体

​ ALL

​ 处理所有的类型请求
