Node 简介

Node.js​

  • 运行在服务器端的 js

  • 用来编写服务器

  • 特点： - 单线程、异步、非阻塞 - 统一 API

    node.js 和 JavaScript 有什么区别

    ECMAScript（node 有） DOM（node 没有） BOM（node 没有）

使用安装工具 Nvm​

除了直接安装外，也可以通过安装工具来安装，使用安装工具安装后更方便我们在不同的 node 版本之间进行切换，使用起来更加灵活。 这里我们以 window 下的 nvm 为例来演示，首先点击 安装 nvm， 下载最新版的 nvm-setup.exe。根据提示下一步下一步即可，同样推荐安装到默认路径。在命令行中输入 nvm version 后，出现版本即表示安装成功。

安装 node​

输入 nvm install latest 下载并安装最新版的 node

输入 nvm use latest 切换到最新版 node，输入 nvm use lts 切换到稳定版 node，也可以输入版本号来切换到指定版本。

Http 服务器​

// 引入内置模块 http
let http = require("http");
let server = http.createServer((req, res) => {
  res.setHeader("Content-Type", "text/html;charset=utf-8");
  //服务器给浏览器的内容
  res.write("hello world");
  //告诉浏览器返还结束了
  res.end();
});
// 设置端口号
server.listen(8989);

Node 模块化规范​

在 node 中，默认支持的模块化规范叫做 CommonJS。 在 CommonJS 中，一个 js 文件就是一个模块

定义模块​

在定义模块时，模块中的内容默认是不能被外部看到的
可以通过 exports 来设置要向外部暴露的内容

访问 exports 的方式有两种：

exports ​

module.exports​

  • 当我们在其他模块中引入当前模块时，require 函数返回的就是 exports
  • 可以将希望暴露给外部模块的内容设置为 exports 的属性
// 可以通过exports 一个一个的导出值
exports.a = "孙悟空";
exports.b = { name: "白骨精" };
exports.c = function fn() {
  console.log("哈哈");
};

// 也可以直接通过module.exports同时导出多个值
module.exports = {
  a: "哈哈",
  b: [1, 3, 5, 7],
  c: () => {
    console.log(111);
  },
};

引入模块​

  • 使用 require("模块的路径")函数来引入模块
    • 引入自定义模块时
    • 模块名要以./ 或 ../开头 - 扩展名可以省略
      在 CommonJS 中，如果省略的 js 文件的扩展名 node，会自动为文件补全扩展名./m1.js 如果没有 js 它会寻找 ./m1.json js --> json --> node（特殊）
  • 引入核心模块时
    • 直接写核心模块的名字即可
    • 也可以在核心模块前添加 node:
const m1 = require("./m1");
// const path = require("path")
const path = require("node:path");

const m2 = require("./m2.cjs");

const hello = require("./hello"); // ./hello/index.jsx

原理​

所有的 CommonJS 的模块都会被封装到一个函数中

(function (exports, require, module, __filename, __dirname) {
  模块代码会被放到这里;
});

console.log(__filename); // __filename表示当前模块的绝对路径

console.log(__dirname); // 当模块所在目录的路径

ES 模块化规范​

默认情况下，node 中的模块化标准是 CommonJS

要想使用 ES 的模块化，可以采用以下两种方案

  1. 使用 mjs 作为扩展名
  2. 修改 package.json 将模块化规范设置为 ES 模块
    当我们设置 "type": "module" 当前项目下所有的 js 文件都默认为 es module
// 导入m4模块，es模块不能省略扩展名（官方标准）
import { a, b, c } from "./m4.mjs";

// 通过as来指定别名
import { a as hello, b, c } from "./m4.mjs";

// 开发时要尽量避免import * 情况
import * as m4 from "./m4.mjs";

// console.log(m4.c)

// 导入模块的默认导出
// 默认导出的内容，可以随意命名
import sum, { a } from "./m4.mjs";

核心模块​

  • 核心模块，是 node 中自带的模块，可以在 node 中直接使用
  • window 是浏览器的宿主对象 node 中是没有的
  • global 是 node 中的全局对象，作用类似于 window
  • ES 标准下，全局对象的标准名应该是 globalThis

缓冲区 buffer​

二进制数据结构

  • 创建 buffer
let b = Buffer.alloc(10);
console.log(b);
  • 可以把字符串转换成 buffer
b = Buffer.from("大家好");
console.log(b);
  • 更快速的创建 buffer
b = Buffer.allocUnsafe(10);
console.log(b);
b.fill(1);
console.log(b);
  • 将 buffer 转换为字符串
b = Buffer.from("大家好");
console.log(b);
var res = b.toString();
console.log(res);

全局变量 process​

  • 表示当前的 node 进程

  • 通过该对象可以获取进程的信息，或者对进程做各种操作

  • 如何使用

    process 是一个全局变量，可以直接使用
    process.exit()

    • 结束当前进程，终止 node

    process.nextTick(callback[, …args])

    • 将函数插入到 tick 队列中
    • tick 队列中的代码，会在下一次事件循环之前执行 会在微任务队列和宏任务队列中任务之前执行
setTimeout(() => {
  console.log(1); // 宏任务队列
});

queueMicrotask(() => {
  console.log(2);
}); // 微任务队列

Promise.resolve().then((result) => {
  console.log(5);
}); //微任务队列

process.nextTick(() => {
  console.log(3); // tick队列
});

console.log(4); // 调用栈

nodejs 和路径有关的预定义变量
process.argv

  • 第一个参数是 nodejs 执行文件的路径地址
  • 第二个参数是执行文件的路径地址
console.log(process.argv);
  • 第一项可以简写
console.log(process.argv0);
console.log(process.argv[1]);

process.cwd() 获取命令执行的路径

console.log(process.cwd());

2 个路径的全局变量
__filename 执行文件的路径地址
__dirname 执行文件上层目录的路径

内置模块 path​

path.resolve（）​

  • 用来生成一个绝对路径 -在计算本地
  • 如果直接调用 resolve，则返回当前的工作目录e:\Download\vs code\code
  • 注意，我们通过不同方式执行代码时，它的工作目录有可能发生变化的
  • 如果将一个相对路径作为参数，
    则 resolve 会自动将其转换为绝对路径
    此时根据工作目录的不同，它所产生的绝对路径也不同
  • 一般会将一个绝对路径作为第一个参数，
    一个相对路径作为第二个参数
    这样它会自动计算出最终的路径
const path = require("node:path");
//以后使用路径时，尽量通过path.resolve（）来生成路径
const result = path.resolve(__dirname, "./hello.js");
console.log(result);

path.join()​

  • 合并处理多个路径
  • 不会生成一个绝对路径
const path = require("path");
const myurl = path.join("./test", "./myindex/../mytest//hello", "index.html");
console.log(myurl);

path.dirname()​

  • 可以获取路径中到文件夹部分的内容
const path = require("path");
myurl = "http://www.baidu.com/index/test/index.jsx";
let res = path.dirname(myurl);
console.log(res);

path.extname()​

  • 获取路径里的文件后缀
const path = require("path");
myurl = "http://www.baidu.com/index/test/index.jsx";
res = path.extname(myurl);
console.log(res);

内置模块 url​

url.parse​

  • 获取 url 里的 querystring 参数
  • url.parse 提供第二个参数，第二个参数可以把 querystring 字符串转成对象
res = url.parse(myurl);
myurl = "http://localhost:3000/index/test/index.jsx?name=zhangsan&age=20#mystr";
res = url.parse(myurl, true);
console.log(res);
console.log(res.query);

fs（File System）​

  • fs 用来帮助 node 来操作磁盘中的文件
  • 文件操作也就是所谓的 I/O， input output
  • 使用 fs 模块，同样需要引用

引入 fs 模块​

const fs = require("node:fs/promises");

文件操作​

fs.writeFile()​

  • 创建一个文件

  • 异步创建一个文件

    fs.writeFile("./1.txt", "我是写入的文字内容", (err) => {
      if (err) {
        return console.log(err);
      }
      console.log("写入成功");
    });
    console.log("同步代码会先执行");

  • 同步创建一个文件

    try {
      fs.writeFileSync("./1.txt", "我是写入的文字内容");
      console.log("创建成功");
    } catch (err) {
      console.log(err);
    }

fs.readFile()​

  • 读取一个文件

  • 异步读取文件

    fs.readFile("./1.txt", (err, data) => {
      if (err) {
        return console.log(err);
      }
      console.log(data.toString());
    });

  • 同步读取文件

    try {
      var data = fs.readFileSync("./1.txt");
      console.log(data.toString());
    } catch (err) {
      console.log(err);
    }

fs.unlink()​

  • 删除文件

  • 不能重复删除

  • 代码删除的内容不会进入回收站； 小心 把重要的代码备份，或者多提交 git

  • 异步删除文件

    fs.unlink("./1.txt", (err) => {
      if (err) {
        return console.log(err);
      }
      console.log("删除成功");
    });

  • 同步删除文件

    try {
      fs.unlinkSync("./1.txt");
      console.log("删除成功");
    } catch (err) {
      console.log(err);
    }

fs.rename()​

  • 修改文件名

  • 异步修改文件名

    fs.rename("./1.html", "./2.html", (err) => {
      if (err) {
        return console.log(err);
      }
      console.log("修改成功");
    });

  • 同步修改文件名

    try {
      fs.rename("./1.html", "./2.html");
      console.log("修改成功");
    } catch (err) {
      console.log(err);
    }

fs.exists()​

  • 判断某个文件是否存在

  • 同步判断

    if (fs.existsSync("./1.txt")) {
      // 如果重复执行就会报错
      fs.unlinkSync("./1.txt");
    } else {
      console.log("文件已经被删除了");
    }

文件夹操作​

fs.mkdir()​

  • 创建一个目录

  • 异步创建一个目录

    fs.mkdir("./test", 0777, function (err) {
      // 第二个参数：第一位是八进制 第二位是所有者权限 第三位是组权限 第四位是所有人
      // 7 满权限 7=1（可执行）+2（可写）+4（可读）
      if (err) {
        return console.log(err);
      }
      console.log("创建成功");
    });

  • 同步创建一个目录

    try {
      fs.mkdirSync("./mytest", 0777);
      console.log("创建成功");
    } catch (err) {
      console.log(err);
    }

fs.remove()​

  • 删除一个目录
  • 注意：删除目录只能删除空目录
  • 删除之后不进入回收站，直接删除
  • 异步删除一个目录
    fs.rmdir("./test", 0777, function (err) {
      if (err) {
        return console.log(err);
      }
      console.log("删除成功");
    });
  • 同步删除一个目录
    try {
      fs.rmdirSync("./mytest", 0777);
      console.log("删除成功");
    } catch (err) {
      console.log(err);
    }

fs.rename()​

  • 修改目录的名字
  • 异步修改
    fs.rename("./mytest", "./mytest2", (err) => {
      if (err) {
        return console.log(err);
      }
      console.log("修改成功");
    });
  • 同步修改
    try {
      fs.renameSync("./mytest", "./mytest2");
      console.log("修改成功");
    } catch (err) {
      console.log(err);
    }

fs.exist()​

  • 判断某个目录是否存在

  • 如果没有 uploads 目录那么就创建 uploads 目录出来

    if (!fs.existSync("./uploads")) {
      try {
        fs.mkdirSync("./uploads", 0777);
      } catch (err) {
        console.log(err);
      }
    } else {
      console.log("已经创建过了");
    }

fs.stat()​

  • 获取该路径的详细信息
  • 判断某个文件是不是目录
    // 1.获取该路径的详细信息
    let stat = fs.statSync(urlstr);
    console.log(stat);
    // 2.通过详细信息对象判断是否是一个文件夹
    if (stat.isDirectory()) {
      console.log("是文件夹或者是目录,Directory");
    } else {
      console.log("是文件,file");
    }
