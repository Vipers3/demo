webpack

Webpack 是一个模块打包器 (bundler)，本质就是一个工具包。

在 Webpack 看来，项目源码中所有的静态资源文件都是模块，如 JS / JSON / CSS / LESS / IMG / FONTS 等等，这些文件都会作为模块来处理。

Webpack 会根据模块之间的依赖关系，进行分析打包所有在整个依赖图中的模块资源，生成最终的浏览器能高效运行的静态资源。

五个核心概念​

Webpack 有五个核心的概念需要知道，分别是

  • Entry 入口
  • Output 出口/输出
  • Mode 模式
  • Loader 加载器
  • Plugin 插件

Entry 入口​

入口起点(entry point) 指示 webpack 应该使用哪个模块，来作为构建其内部依赖图的开始。进入入口起点后，webpack 会找出有哪些模块和库是入口起点依赖的 ​

Output 出口​

Output 属性告诉 webpack 在哪里生成(输出)打包文件(bundles)，以及如何命名打包文件

Mode 模式​

Webpack 主要有两种工作模式，分别是 development 开发模式和 production 生产模式, 不同模式提供了不同的默认配置。

Loader 加载器​

Webpack 默认只能处理 JS / JSON 文件，如果需要打包其他类型文件，就需要使用对应的 loader 进行处理。例如 css-loader，less-loader，babel-loader 等等

Loader 本质是 JS 函数，接受源文件内容，返回转换后的结果内容 ​

Plugin 插件​

loader 用于转换某些类型的模块，而插件则可以用于执行范围更广的其它任务。包括：压缩文件，代码规范检查等

webpack 使用​

​ 在使用webpack之前，先安装插件

​ $ npm i webpack webpack-cli -D

webpack 直接使用​

​ $ npx webpack --mode production/development

​ 最终生成一个dist目录，把js直接汇总到一个文件中，并且如果是生产模式还会自动压缩。

$ npx webpack 
    --entry ./src/main.js 
    --output-path ./build/js 
    --output-filename bundle.js 
    --mode development

​ 指定入口文件，指定输出的目录和文件名称，指定模式

使用配置文件的方式​

/*
    a. webpack 默认只能打包js或者json文件
    b. webpack 对于 es6+ 的内容并没有处理
    c. webpack 指令式语法自定义目录或者文件名称比较复杂
*/

// 使用 webpack 的配置文件，能够更加详细的控制项目的构建方式
// 5 项核心配置内容(入口，输出，模式，loaders，插件)
const path = require('path');

// webpack 底层原理还是使用的nodejs，所以配置文件的模块化规范是commonjs
module.exports = {
    // 入口文件
    entry: './src/index.jsx',

    // 输出目录和文件
    output: {
        path: path.resolve(__dirname, './dist/js'),
        filename: 'bundle.js'
    },

    // 模式  production   development
    mode: 'production'
}

​ 配置完成之后使用 $ npx webpack 运行项目

webpack 配置​

​ webpack 只能解析 JavaScript 和 JSON 文件，这是 webpack 开箱可用的自带能力。loader 让 webpack 能够去处理其他类型的文件。

loaders​

​ 在入口文件中，加载css文件，默认会报错

import './css/reset.css';

​ 通过以下配置完成

​ $ npm i css-loader style-loader -D

module.exports = {
    
    // 4. loaders 用于处理非js的模块 - 加载规则是从右往左
    module: {
        // 这里处理模块无非就是在声明一些规则
        rules: [
            // 4.1 处理css
            { test: /\.css$/, use: ['style-loader' ,'css-loader'] }
        ]
    }
}

处理less​

​ $ npm i less less-loader -D

​ 引入less文件

import './css/index.less';

​ 配置解析规则

module.exports = {
    // 4. loaders 用于处理非js的模块 - 加载规则是从右往左
    module: {
        // 这里处理模块无非就是在声明一些规则
        rules: [
            // 4.2 处理less
            { test: /\.less$/, use: ['style-loader' ,'css-loader', 'less-loader'] }
        ]
    }
}

​ less 片段

处理JavaScript​

​ $ npm i babel-loader @babel/core @babel/preset-env -D

​ 编辑es6 的内容， 会完全把es6转换为es5，但是es6的部分语法还是不支持的

// 目的是在ie浏览器中测试的时候，依然生效，不要报错
const arr1 = new Set(['aaa', true, 100, 'hello', 'aaa']);
console.log(arr1);

​ 声明配置

// 4.3 处理高级的JavaScript
{
    test: /\.js$/,
    // 排除 node_modules
    exclude: /node_modules/,
    // use 除了直接写数组之外，也可以分别指定loader和插件
    use: {
        loader: 'babel-loader',
        // 其他配置
        options: {
            // 预设方式，选择固定的场景解析es6的代码
            presets: ['@babel/preset-env'],
            // 指定使用的插件
            plugins: []
        }
    }
}

​ 另外，为了照顾更多的浏览器，需要在 package.json 中添加字段

"browserslist": [
  "> 0.1%",
  "not ie <=8"
]

处理JavaScript的兼容​

​ 下载相关的配置项 - $ npm i @babel/polyfill -D

​ 声明处理背景图片的规则

// 4.4 处理背景图片
{
    test: /\.(jpg|png|gif|webp|jpeg|bmp)$/,
    // 指定类型
    type: 'asset',
    // 指定转换方式
    parser: {
        // 指定文件的大小范围, 如果图片在规定大小之内则会生产为base64格

完整配置文件​

webpack.config.js

/*
    a. webpack 默认只能打包js或者json文件
    b. webpack 对于 es6+ 的内容并没有处理
    c. webpack 指令式语法自定义目录或者文件名称比较复杂
*/

// 使用 webpack 的配置文件，能够更加详细的控制项目的构建方式
// 5 项核心配置内容(入口，输出，模式，loaders，插件)
const path = require('path');
const HTMLWebpackPlugin = require('html-webpack-plugin');
const MiniCSSExtractPlugin = require('mini-css-extract-plugin');

// webpack 底层原理还是使用的nodejs，所以配置文件的模块化规范是commonjs
module.exports = {
    // 1. 入口文件
    entry: './src/index.jsx',

    // 2. 输出目录和文件
    output: {
        path: path.resolve(__dirname, './dist'),
        filename: './js/bundle.js',
        // 清除之前的内容
        clean: true
    },

    // 3. 模式  production   development
    mode: 'development',

    // 4. loaders 用于处理非js的模块 - 加载规则是从右往左
    module: {
        // 这里处理模块无非就是在声明一些规则
        rules: [
            // 4.1 处理css
            { 
                test: /\.css$/, 
                use: [
                    MiniCSSExtractPlugin.loader ,
                    'css-loader',
                    'postcss-loader'
                ] 
            },

            // 4.2 处理less
            { 
                test: /\.less$/, 
                use: [
                    MiniCSSExtractPlugin.loader ,
                    'css-loader', 
                    'postcss-loader',
                    'less-loader'
                ]
            },

            // 4.3 处理高级的JavaScript
            {
                test: /\.js$/,
                // 排除 node_modules
                exclude: /node_modules/,
                // use 除了直接写数组之外，也可以分别指定loader和插件
                use: {
                    loader: 'babel-loader',

                    // 其他配置
                    options: {
                        // 预设方式，选择固定的场景解析es6的代码
                        presets: [
                            [
                                '@babel/preset-env',
                                {
                                    // 只打包使用的ES6新API的实现代码
                                    useBuiltIns: 'usage',
                                    // 指定core-js的版本号为2
                                    corejs: { version: 2 }
                                }
                            ]
                        ],
                        // 指定使用的插件
                        plugins: []
                    }
                }
            },

            // 4.4 处理背景图片
            {
                test: /\.(jpg|png|gif|webp|jpeg|bmp)$/,
                // 指定类型
                type: 'asset',
                // 指定转换方式
                parser: {
                    // 指定文件的大小范围, 如果图片在规定大小之内则会生产为base64格式的图片
                    dataUrlCondition: {
                        // 以kb作为结算单位
                        maxSize: 10 * 1024
                    }
                },
                // 如果超出大小，指定生成的文件名称和路径
                generator: {
                    // 指定渲染的路径，指定名称和随机字符，保留原来的后缀名
                    filename: './imgs/bg-[hash:8][ext]'
                }
            },

            // 4.5 处理字体图标
            {
                test: /\.(ttf|woff|woff2)$/,
                // 指定类型
                type: 'asset',
                // 指定转换方式
                parser: {
                    // 指定文件的大小范围, 如果图片在规定大小之内则会生产为base64格式的图片
                    dataUrlCondition: {
                        // 以kb作为结算单位
                        maxSize: 8 * 1024
                    }
                },
                // 如果超出大小，指定生成的文件名称和路径
                generator: {
                    // 指定渲染的路径，指定名称和随机字符，保留原来的后缀名
                    filename: './fonts/font-[hash:6][ext]'
                }
            }
        ]
    },
    
    // 5. 插件 plugins
    plugins: [
        new HTMLWebpackPlugin({
            // 指定模板文件
            template: './public/index.html',
            // 指定哈希
            hash: true
        }),

        // 提取css作为外链的插件声明
        new MiniCSSExtractPlugin({
            filename: './css/[hash:8].css'
        })
    ],


    // 配置虚拟服务，启动自动化构建的方式
    devServer: {
        // 指定端口
        port: 8080,
        // 自动打开
        open: true,
        hot: true
    }
}
