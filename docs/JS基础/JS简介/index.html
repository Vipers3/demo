2、导入 JS 的方式​

行内样式​

<button onclick="alert('你点我干啥')">点我一下</button>
<!-- 不好看，可读性差，容易出现引号嵌套错误，形内插入js一般不用 -->

内部样式​

<script type="text/javascript">alert("哈哈哈")</script>
  • 内部嵌入 js 用 script
  • html 网页的任何位置 js 都可以运行
  • script 标签的 type 属性可以省略

外部引入​

<script scr="./index.js"></script>
/* 导入外部js文件 */

3、JS 的执行顺序​

--js代码的执行顺序是从上到下按顺序执行的
--js是阻断性语言，在js运行时，其他代码都会停止运行
<!-- 大部分代码都需要内容加载完之后再执行，如果内容没有加载就会报错 -->

延迟执行​

defer

  • 如果在外部导入 script 标签内设置 defer 属性，js 会最后执行
  • 如果有多个 defer 属性，按顺序执行
  • defer 属性只能在外部导入设置，内部不好使
  • 使用场景：如果 js 代码需要依赖其他框架执行，需要先加载框架，后执行代码
<script defer scr="./index.js"></script>

异步加载​

async

  • 外部导入 js 的时候使用，async 是 script 的一个属性
  • async 功能是碰到加载时间特别长的代码，可以设置异步加载，底下的 js 代码就不会阻塞，不然正常情况下 js 代码都是同步加载，碰到比较大的代码就容易阻塞
<script async scr="./index.js"></script>

4、JS 的输出方式​

弹窗​

window.alert(); /*  BOM工具包 里面的弹窗API  */

控制台输出​

window.console.log(); /* BOM工具包  控制台输出API */

在网页的 body 输出内容​

  • 如果在文档流加载完毕之间书写 document.write 方法，是在原来的文档流设置输出内容，在原来的基础上追加新内容
  • 在整个文档流加载完毕之后，再执行 document.write 方法，底层会调用 open（）方法，开辟新的文档流，内容在新的文档流书写，原来的内容就会被覆盖
  • 不支持异步加载，也不支持延迟执行
document.write();

弹出输入框​

prompt("请输入内容");

5、变量​

变量：存储数据的容器（盒子）

存储机制：声明一个变量，就会再内层开辟一个空间，空间存储的就是变量的值，可以通过变量名来获取数据

变量的声明​

let a;
console.log(a); //声明变量未赋值，返回undefined
// 分清变量名和数据
var name = "小红";
console.log(name); //变量
console.log("name"); //字符串
var a = 10;
// 变量重新赋值
a = 20;
var a = 33;
//在ES5 用var定义变量名重复，不会报错会覆盖
console.log(a); //undefined
var a;
/* js在执行代码之前，重新编译一次（预解析） 所有var声明的变量会放在所有的最上面 */

变量的命名规范​

  • 组成：字母，数字，下划线，$组成，开头不能数字开头
  • 大小写敏感：name Name
  • 有意义命名：userName（推荐驼峰命名法）
 var _jack=1
 var $1h=2
 var 1kus=3 //不符合规范

变量的交换​

第一种，设置第三个变量

// a和b的值交换
var a = 10;
var b = 20;
var temp = a;
a = b;
b = temp;

第二种,不用第三个变量

var a = 10;
var b = 20;
a = a + b;
b = a - b;
a = a - b;

第三种 解构赋值

var a = 10;
var b = 20;
[a, b] = [b, a];
